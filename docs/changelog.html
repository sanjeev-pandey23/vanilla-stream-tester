Stream Tester — Changelog

Developer-focused player · Sanjeev Pandey's Lab

v0.0.4 27 Apr 2026
Utility
  • New Share URL — the Share button in the config panel encodes the full current setup (stream URL, type, DRM, playback toggles, custom headers, bitrate cap, ABR algorithm, simulated bandwidth) into URL query parameters. Paste the resulting link in any browser to instantly restore the exact configuration. A Copy button writes the URL to the clipboard via the Clipboard API with a execCommand fallback for older browsers. Only non-default values are included, keeping URLs short. On page load, restoreFromUrl() reads the query string and pre-fills all matching fields automatically.
v0.0.3 27 Apr 2026
Network & ABR Features
  • New HLS DASH Custom Request Headers — inject arbitrary key:value headers into every manifest and segment request. Useful for Authorization, CDN authentication tokens, or any custom header required by the origin. Enter one header per line in Key: Value format. HLS uses xhrSetup; DASH uses RequestModifier.
  • New HLS DASH Max Bitrate Cap — hard ceiling on the quality tier ABR can select. Prevents the player from ever requesting a rendition above the chosen bitrate, regardless of measured throughput. HLS maps the cap to the highest level index ≤ the target via autoLevelCapping; DASH uses streaming.abr.maxBitrate.video.
  • New DASH ABR Algorithm Selector — switch between dash.js ABR strategies without needing devtools. Available options: abrThroughput (default), abrBola (buffer-occupancy based), abrDynamic, abrL2A, and abrLoLP (optimised for low-latency streams). Applied via streaming.abr.ABRStrategy before initialisation.
  • New HLS DASH Simulated Bandwidth — override the player's perceived network throughput to test ABR decisions without throttling the OS or browser. Presets cover 2G / 3G / 4G and broadband speeds. HLS seeds abrEwmaDefaultEstimate and reinforces the value every 2 s; DASH sets initialBitrate and maxBitrate to the simulated value.
v0.0.2 27 Apr 2026
Bug Fixes
  • Fix HLS DASH Stream type detection with query parameters — URLs with anything after the file extension (e.g. ?bc_token=…, SSAI redirect URLs) were not detected as HLS or DASH, causing playback to fail. Detection now extracts the URL pathname via new URL() before checking the extension, stripping query strings and fragments automatically.
